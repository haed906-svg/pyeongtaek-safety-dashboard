06

추가 제안 — 더 집중할 수 있게 만드는 AI 활용 방안

01

안전작업허가 AI 자동 심사 시스템

협력업체 서류 업로드 → AI 항목별 1차 검토 → 누락·불일치 자동 표시 → 담당자 최종 승인. 현재 진행 중인 웹앱(safety6119)에 탭으로 바로 통합 가능.

심사 시간 70% 단축 예상
02

소방설비 이상 자동 알림

72개 건물 점검 이력 DB를 AI가 분석 → 결함 반복 구역, 유효기간 임박 설비 자동 알림. 카카오 알림톡 연동 계획과 결합 가능.

우선순위 점검 집중 가능
03

사고보고서 자동 초안 생성

사고 시각·장소·유형·조치내용 입력 → SHEE Portal 기준 1·2·3차 보고서 초안 자동 완성. 골든타임 내 보고 품질 향상.

초동 보고 시간 단축
04

업무 공수 분석 대시보드

월 투입공수(Hr) 데이터 기반 과부하 업무·인력 집중 시점 시각화. 경영층 인력 효율화 근거 자료. GitHub Pages 배포.

인력 운영 근거 데이터화
05

법정교육 이수 자동 관리

부서별 교육 이수 현황 자동 집계 → 미이수자 알림 → 교육 로드맵 초안 자동 생성. 안전체험관 반복 일정관리 AI 대체.

교육 행정 업무 자동화

핵심 차이점

01

분석 범위

자체 분석은 안전·방재·공통 분야를 구분해 AI 전환 우선순위 도출. 인사노경은 자동화 가능 여부를 즉시·부분·어려움 3단계로 분류.

02

절감 효과 산출

인사노경 분석은 업무별 Hr 절감량을 수치화(총 약 690Hr/월 절감 예상). 자체 분석은 전환 가능 건수·비율 중심으로 정리.

03

종합상황실 주목

인사노경은 종합상황실 24시간 근무(65.5Hr/월 × 5건)를 단순반복 최대 업무로 별도 부각. 자체 분석에서는 Human Only로 분류.

양측 분석이 일치하는 핵심 결론

  • 안전작업허가 심사(12건) — 두 분석 모두 AI 자동화 최우선 대상으로 지목
  • 소방설비 도면·DB 관리 — 즉시 자동화 가능한 반복 문서 업무로 공통 확인
  • 화재진압·인명구조·응급처치 — 물리적 현장 수행으로 자동화 불가 공통 결론
  • 보고서·상황보고 자동 초안 생성 — 양측 모두 AI 보조 핵심 영역으로 선정
  • 위험물 입출고·차량 점검 기록 — 디지털화·자동화로 즉시 효율 개선 가능
